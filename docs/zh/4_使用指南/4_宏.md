# 宏说明
## 特性宏
若使用openHiTLS提供的构建方式，则无需手动设置宏，参考[1_构建及安装指导](1_构建及安装指导.md)，使用`--enable`开启某个特性，或使用`--disable`关闭。

全量的特性宏列表，参考[feature.json](../../config/json/feature.json)： libs/hitls_*/features/\*/

## 密钥生成相关宏
- `CRYPT_DH_TRY_CNT_MAX`
    - 默认值: 100
    - 说明: DH密钥对生成的最大尝试次数。当生成的密钥不满足要求时,会重新生成,直到达到此次数限制。
    - 建议：保持默认值100，除非有特殊性能要求。

- `CRYPT_DSA_TRY_MAX_CNT`
    - 默认值: 100
    - 说明: DSA密钥对生成的最大尝试次数。当生成的密钥不满足要求时,会重新生成,直到达到此次数限制。
    - 建议：保持默认值100，除非有特殊性能要求。

- `CRYPT_ECC_TRY_MAX_CNT`
    - 默认值: 100
    - 说明: ECC密钥对生成的最大尝试次数。当生成的密钥不满足要求时,会重新生成,直到达到此次数限制。
    - 建议：保持默认值100，除非有特殊性能要求。

## 随机数生成相关宏
- `DRBG_MAX_RESEED_INTERVAL`
    - 默认值: 10000
    - 说明: DRBG(确定性随机数生成器)重新播种的最大间隔。每生成10000次随机数后,需要重新获取熵源进行播种。
    - 建议：保持默认值10000，过大会降低随机数安全性，过小会影响性能。

- `ENTROPY_USE_DEVRANDOM`
    - 说明: 使用操作系统提供的设备随机数作为熵源。在Linux系统上通常使用/dev/random或/dev/urandom。

## 系统相关宏
- `HITLS_BIG_ENDIAN`
    - 说明: 指示系统使用大端字节序。影响数据的存储和传输格式。
    - 注意：若未指定，则默认使用小端字节序。

- `HITLS_BSL_LOG_NO_FORMAT_STRING`
    - 说明: 日志输出不使用格式化字符串,直接输出原始字符串。可以提高日志性能。此功能主要被应用于协议模块。
    - 注意：若未指定，则默认使用格式化字符串。
    - 建议：在需要提升日志性能的场景下，启用此宏。

- `HITLS_BSL_SAL_LINUX`
    - 说明: 使用Linux系统抽象层。用于适配Linux操作系统的系统调用。
    - 注意：若未指定，用户需要自行注册SAL接口。
    - 相关特性：sal_mem、sal_thread、sal_lock、sal_file、sal_net、sal_time、sal_dl

- `HITLS_CRYPTO_NO_AUXVAL`
    - 说明: 不使用辅助向量获取CPU特性。需要使用其他方式检测CPU特性。

- `HITLS_CRYPTO_ASM_CHECK`
    - 说明: 启用汇编代码检查。在运行时检查CPU是否支持相应的指令集扩展。当前支持检查的算法有：
        - aes: x8664/armv8
        - sm4: x8664/armv8
        - gcm: x8664/armv8
        - md5: x8664
        - sha1: x8664
        - sha2: x8664
        - sm3: x8664
        - ecc: x8664
    - 仅在ealinit特性开启时生效。

- `HITLS_EAL_INIT_OPTS`
    - 说明: EAL(加密适配层)初始化选项。用于配置加密模块的初始化参数。
    - 用于CRYPT_EAL_Init函数，可替换此函数的入参，取值参考头文件`include/crypto/crypt_eal_init.h`，可组合使用以下值：
        - CRYPT_EAL_INIT_CPU
        - CRYPT_EAL_INIT_BSL
        - CRYPT_EAL_INIT_RAND
        - CRYPT_EAL_INIT_PROVIDER

## 大数相关宏
- `HITLS_SIXTY_FOUR_BITS`、`HITLS_THIRTY_TWO_BITS`
    - 说明: 指示当前系统是64位平台还是32位平台。
    - 默认值：HITLS_SIXTY_FOUR_BITS，即64位模式。
    - 建议：与当前系统位数保持一致。

## 算法相关宏
- `HITLS_CRYPTO_INIT_RAND_ALG`
    - 说明: 初始化随机数算法，用于DRBG的初始化。
    - 默认值：CRYPT_RAND_SHA256
    - 可选值：参考头文件`include/crypto/crypt_algid.h`中的CRYPT_RAND_AlgId

- `HITLS_CRYPTO_NIST_ECC_ACCELERATE`
    - 说明: 使用NIST曲线的硬件加速。此宏**默认开启**，在`config/json/compile.json`中配置。
    - 建议：在支持硬件加速的平台上，启用此宏以提升ECC运算性能。

# 设置宏
参考[1_构建及安装指导](1_构建及安装指导.md)，使用`--add_options`添加宏或`--del_options`删除宏。

示例：
```bash
python3 ../configure.py --add_options="-DHITLS_CRYPTO_NIST_ECC_ACCELERATE"
```
