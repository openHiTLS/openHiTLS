# RFC 9150：TLS 1.3 仅完整性套件（TLS_SHA256_SHA256 / TLS_SHA384_SHA384）

本文说明如何在 openHiTLS 中**编译、配置与验证** [RFC 9150](https://www.rfc-editor.org/rfc/rfc9150.html) 定义的 TLS 1.3 仅完整性密码套件，并指向仓库内的**自动化测试**与**示例程序**。

## 1. CMake 选项

RFC 9150 记录层依赖 **HMAC** 与 **TLS 1.3**，并由套件推导逻辑在满足依赖时自动打开 `TLS_SHA256_SHA256` / `TLS_SHA384_SHA384`：

- `HITLS_TLS_SUITE_CIPHER_TLS13_INTEGRITY=ON`（核心开关）
- 同时需要：`HITLS_TLS_SUITE_KX_ECDHE`、至少一种 `HITLS_TLS_SUITE_AUTH_*`（如 RSA/ECDSA/PSK）、`HITLS_TLS_PROTO_TLS13=ON`

使用 **full** 配置并额外打开完整性套件时，典型命令为：

```bash
cd openHiTLS
git submodule update --init platform/Secure_C
mkdir -p build && cd build
cmake .. -DHITLS_BUILD_PROFILE=full -DHITLS_TLS_SUITE_CIPHER_TLS13_INTEGRITY=ON -DHITLS_BUILD_GEN_INFO=ON
make -j
```

配置成功后，`build/macros.txt` 中应出现 `-DHITLS_TLS_SUITE_TLS_SHA256_SHA256`、`-DHITLS_TLS_SUITE_TLS_SHA384_SHA384` 等宏。

## 2. 自动化测试（SDV）

在**已完成主库编译**且 `build/macros.txt` 存在的前提下，仅构建并运行 RFC 9150 相关用例：

```bash
cd testcode
bash script/build_sdv.sh 'run-tests=test_suite_sdv_tls_rfc9150|test_suite_sdv_frame_tls_rfc9150' no-demos
cd output
./test_suite_sdv_tls_rfc9150
./test_suite_sdv_frame_tls_rfc9150
```

说明：

- `test_suite_sdv_tls_rfc9150`：套件信息、`HITLS_CFG_SetCipherSuites` 等。
- `test_suite_sdv_frame_tls_rfc9150`：FRAME 模拟链路下的 TLS 1.3 握手、应用数据往返、`TLS_SHA384_SHA384` 端到端，以及**篡改 MAC 后**的 `bad_record_mac` 负例。

## 3. 示例程序（demo）

源码位于 [testcode/demo/rfc9150_server.c](../../../testcode/demo/rfc9150_server.c) 与 [testcode/demo/rfc9150_client.c](../../../testcode/demo/rfc9150_client.c)。要点：

- 使用 `HITLS_CFG_NewTLS13Config()`；
- 仅配置 `HITLS_TLS_SHA256_SHA256`（端口 **12346**，避免与默认 demo 的 12345 冲突）。

编译（需传入与主库一致的 `CUSTOM_CFLAGS`，通常即 `build/macros.txt` 内容）：

```bash
cd testcode/demo
rm -rf build && mkdir build && cd build
CUSTOM_CFLAGS=$(tr '\n' ' ' < ../../../build/macros.txt)
cmake -DCUSTOM_CFLAGS="$CUSTOM_CFLAGS -D__FILENAME__=__FILE__" ..
make -j rfc9150_server rfc9150_client
```

运行顺序：先启动服务端，再启动客户端。

```bash
./rfc9150_server
# 另一终端
./rfc9150_client
```

证书路径与现有 `client.c`/`server.c` 相同，使用仓库内 `testcode/testdata/tls/certificate/der/ecdsa_sha256/` 下测试证书。

## 4. 安全提示

RFC 9150 套件**不提供机密性**；仅适用于已明确接受明文传输风险的场景。部署前需完成威胁建模与合规评估。
