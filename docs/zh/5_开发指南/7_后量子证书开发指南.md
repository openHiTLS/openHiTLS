#     openHiTLS 后量子证书 开发指南

## 概述

随着量子计算技术的快速发展，传统公钥密码体系在理论上面临被量子算法（如 Shor 算法）破解的风险。为应对这一挑战，后量子密码学（Post-Quantum Cryptography, PQC）逐渐成为密码标准化和产业实践的重点方向。openHiTLS 项目自 0.3.0 版本中重点构筑后量子密码算法、协议及配套PKI体系，全面支持NIST 标准化后量子算法，实现了后量子技术从算法层到应用层的完整落地。本文档旨在系统梳理 openHiTLS 中后量子证书与密钥相关功能的使用方式，详细说明核心 API 的调用流程、参数含义、返回值说明及典型错误场景，并提供可参考的伪代码示例，帮助开发者快速完成集成与验证工作。

## 后量子能力支持概览

openHiTLS在新版本中支持的PQC算法主要分为以下几类：

### 后量子数字签名算法

+ **ML-DSA**：支持符合NIST后量子密码标准的 ML-DSA，具体包括ML-DSA-44、ML-DSA-65、ML-DSA-87；
+ **SLH-DSA**：支持基于哈希的无状态签名算法SLH-DSA，具体包括SLH-DSA的SHA2/SHAKE族在128/192/256安全等级下的Small与Fast两类参数集；
+ **XMSS**：支持基于 Merkle 树的状态型哈希签名算法，具体包括XMSS与XMSSMT的SHA2、SHAKE、SHAKE256 变种，在不同树高度、层数及安全参数下的全部标准化参数组合。

### 后量子密钥封装算法

+ **ML-KEM**：支持 ML-KEM-512、ML-KEM-768、ML-KEM-1024 三种标准参数集；
+ **FrodoKEM**：支持 FrodoKEM-640、FrodoKEM-976、FrodoKEM-1344参数集，并提供 SHAKE 与 AES 两种内部伪随机生成变种；
+ **Classic McEliece**：支持 mceliece348864、mceliece460896以及 mceliece6688128、mceliece6960119、mceliece8192128等标准编码参数集。

### 混合后量子算法

+ **X25519 + ML-KEM**：支持 X25519 与 ML-KEM-512/768/1024 组合的混合密钥协商模式，在保持经典椭圆曲线安全性的同时提供对应 NIST Level 1/3/5 的后量子安全等级；
+ **ECDH + ML-KEM**：支持基于传统 ECDH与 ML-KEM-512/768/1024 的混合密钥协商机制，实现经典密码体系与后量子密码体系的双重安全保障。

openHiTLS在X.509证书体系中支持以下PQC算法作为证书的公钥/签名算法：

- **ML-DSA**
- **SLH-DSA**
- **XMSS**
- **ML-KEM**

> 说明：FrodoKEM 与 Classic McEliece 在当前实现中主要用作密钥封装，不作为主流证书签名算法。

## 密钥相关接口

### 密钥上下文控制

#### **`CRYPT_EAL_PkeyNewCtx`**

- **描述**：根据指定的算法ID创建并初始化一个非对称密钥上下文结构`CRYPT_EAL_PkeyCtx`。

- **函数原型**： 

  ```c
  CRYPT_EAL_PkeyCtx *CRYPT_EAL_PkeyNewCtx(CRYPT_PKEY_AlgId id)
  ```

- **参数描述**：

  - `id`：表示非对称加密算法标识，用于指定非对称密钥算法类型的算法标识符，函数根据该算法类型创建对应的密钥上下文结构。

- **返回值**： 指向新创建的`CRYPT_EAL_PkeyCtx` 非对称密钥上下文结构的指针，创建失败则返回`NULL`。

#### **`CRYPT_EAL_PkeySetParaById`**

- **描述**：该函数根据指定的参数 ID，为指定的非对称密钥上下文设置对应的算法参数。

- **函数原型**： 

  ```c
  int32_t CRYPT_EAL_PkeySetParaById(CRYPT_EAL_PkeyCtx *pkey, CRYPT_PKEY_ParaId id)
  ```

- **参数描述**：

  - `pkey`：表示非对称密钥上下文结构指针，用于指定要设置参数的密钥对象，函数将在该结构中写入或更新对应的算法参数。
  - `id`：表示非对称加密算法参数标识，用于指定要设置的参数类型，函数根据该参数标识设置对应的算法参数。

- **返回值**：返回`HITLS_PKI_SUCCESS`。

#### **`CRYPT_EAL_PkeyGen`**

- **描述**：通过当前配置的密钥上下文生成非对称密钥对，并将生成的密钥数据保存回上下文结构中，为后续的加密、签名等操作做好准备。

- **函数原型**： 

  ```c
  int32_t CRYPT_EAL_PkeyGen(CRYPT_EAL_PkeyCtx *pkey)
  ```

- **参数描述**：

  - `pkey`：表示非对称密钥上下文结构指针，用于接收生成的密钥数据。函数会根据此上下文的配置信息生成密钥对，并将密钥存储在该结构内。

- **返回值**：返回`HITLS_PKI_SUCCESS`。

#### **`CRYPT_EAL_PkeySetPub`**

- **描述**：将外部提供的公钥数据设置到密钥上下文中，确保上下文结构可以用于后续的加密、验签等操作。调用者需要自行申请和释放公钥数据内存。

- **函数原型**： 

  ```c
  int32_t CRYPT_EAL_PkeySetPub(CRYPT_EAL_PkeyCtx *pkey, const CRYPT_EAL_PkeyPub *key)
  ```

- **参数描述**：

  - `pkey`：表示非对称密钥上下文结构指针，用于接收公钥数据。函数会将公钥数据设置到该结构中，以便后续的密钥对操作。
  - `key`：表示公钥数据指针，将该公钥数据存入非对称密钥上下文结构指针 `pkey` 中。

- **返回值**：返回`HITLS_PKI_SUCCESS`。

#### **`CRYPT_EAL_PkeySetPrv`**

- **描述**：将外部提供的私钥数据设置到密钥上下文中，确保上下文结构可以用于后续的解密、签名等操作。调用者需要自行申请和释放私钥数据内存。

- **函数原型**： 

  ```c
  int32_t CRYPT_EAL_PkeySetPrv(CRYPT_EAL_PkeyCtx *pkey, const CRYPT_EAL_PkeyPrv *key)
  ```

- **参数描述**：

  - `pkey`：表示非对称密钥上下文结构指针，用于接收私钥数据。函数会将私钥数据设置到该结构中，以便后续的密钥对操作。
  - `key`：表示私钥数据指针，将该私钥数据存入非对称密钥上下文结构指针 `pkey` 中。

- **返回值**：返回`HITLS_PKI_SUCCESS`。

#### **`CRYPT_EAL_PkeyCtrl`**

- **描述**：用于处理非对称密钥上下文的通用接口，通过传入不同的操作枚举值`opt`和`void`类型指针`val`实现对密钥上下文的统一控制。

- **函数原型**： 

  ```c
  int32_t CRYPT_EAL_PkeyCtrl(CRYPT_EAL_PkeyCtx *pkey, int32_t opt, void *val, uint32_t len)
  ```

- **参数描述**：

  - `pkey`：表示非对称密钥上下文结构指针，用于指定要操作的密钥会话，根据不同的`opt`和`val`对密钥上下文执行操作。
  - `opt`：表示选项标识符，用于指定本次调用执行的具体操作类型，函数内部根据该命令进行分支调度。
  - `val`：通用数据指针，用于传入或传出具体数据，函数内部根据`cmd`进行数据类型的转换。
  - `len`：表示`val`所指向的待操作数据的长度。

- **返回值**：返回`HITLS_PKI_SUCCESS`。

#### **`CRYPT_EAL_PkeyEncapsInit`**

- **描述**：用于初始化非对称密钥封装上下文，并根据指定的算法参数进行配置，常用于后量子密码学算法中以ML-KEM为代表的密钥封装机制。

- **函数原型**： 

  ```c
  int32_t CRYPT_EAL_PkeyEncapsInit(CRYPT_EAL_PkeyCtx *pkey, const BSL_Param *params)
  ```

- **参数描述**：

  - `pkey`：表示非对称密钥上下文结构指针，用于初始化密钥封装上下文，存储封装过程中的密钥数据。
  - `params`：后量子算法的参数结构体，包含初始化所需的算法参数，如密钥封装方案、加密模式等。

- **返回值**：返回`HITLS_PKI_SUCCESS`。

#### **`CRYPT_EAL_PkeyEncaps`**

- **描述**：执行密钥封装操作，通过加密算法将共享密钥封装在密文中，利用给定的密钥上下文生成加密后的密文和共享密钥。

- **函数原型**： 

  ```c
  int32_t CRYPT_EAL_PkeyEncapsInit(CRYPT_EAL_PkeyCtx *pkey, const BSL_Param *params)
  ```

- **参数描述**：

  - `pkey`：已初始化的非对称密钥上下文结构指针，包含密钥封装操作所需的密钥数据和算法配置。
  - `cipher`：输出缓冲区，用于存放封装后的密文。
  - `cipherLen`：输入缓冲区的容量，输出实际生成的密文长度。
  - `sharekey`：输出缓冲区，用于存放封装过程生成的共享密钥。
  - `shareKeyLen`：输入缓冲区的容量，输出实际生成的共享密钥长度。

- **返回值**：返回`HITLS_PKI_SUCCESS`。

#### **`CRYPT_EAL_PkeyDecapsInit`**

- **描述**：初始化非对称密钥解封装上下文，根据给定的算法参数配置，用于后量子密码学中的密钥解封装操作，恢复共享密钥。

- **函数原型**： 

  ```c
  int32_t CRYPT_EAL_PkeyDecapsInit(CRYPT_EAL_PkeyCtx *pkey, const BSL_Param *params)
  ```

- **参数描述**：

  - `pkey`：非对称密钥上下文结构指针，指定要进行解封装操作的密钥上下文。
  - `params`：后量子算法的参数结构体，包含解封装操作所需的算法参数，如密钥解封装方案等

- **返回值**：返回`HITLS_PKI_SUCCESS`。

#### **`CRYPT_EAL_PkeyDecaps`**

- **描述**：执行密钥解封装操作，在后量子密码学的密钥封装机制中，根据输入的封装密文恢复共享密钥，用于安全通信中的密钥协商过程。

- **函数原型**： 

  ```c
  int32_t CRYPT_EAL_PkeyDecaps( const CRYPT_EAL_PkeyCtx *pkey, const uint8_t *cipher, uint32_t cipherLen, uint8_t *sharekey, uint32_t *shareKeyLen)
  ```

- **参数描述**：

  - `pkey`：已初始化的非对称密钥上下文结构指针，包含解封装操作所需的私钥及算法配置信息。
  - `cipher`：输入的封装密文数据缓冲区，用于提供需要解封装的密文内容。
  - `cipherLen`：输入密文数据的长度，单位为字节。
  - `sharekey`：输出缓冲区，用于存放解封装后恢复得到的共享密钥。
  - `shareKeyLen`：输入时表示共享密钥缓冲区的容量；输出时表示实际恢复的共享密钥长度。

- **返回值**：返回`HITLS_PKI_SUCCESS`。

### 密钥解析与编码

#### **`CRYPT_EAL_DecodeBuffKey`**

- **描述**：用于解析指定格式的非对称密钥编码缓冲区并创建对应的密钥上下文结构，在后量子密码算法中常用于加载和恢复后量子公钥或私钥数据，以便进行后续的签名、验签或密钥封装操作。

- **函数原型**： 

  ```c
  int32_t CRYPT_EAL_DecodeBuffKey(int32_t format, int32_t type, BSL_Buffer *encode, const uint8_t *pwd, uint32_t pwdlen, CRYPT_EAL_PkeyCtx **ealPKey)
  ```

- **参数描述**：

  - `format`：输入参数，指定密钥编码缓冲区的格式类型，当前支持传入枚举值`BSL_FORMAT_UNKNOWN`（自动判别格式）、`BSL_FORMAT_PEM`（PEM格式）或`BSL_FORMAT_ASN1`（DER格式）。
  - `type`：输入参数，指定密钥类型，表示待解码的非对称密钥类别，由`CRYPT_ENCDEC_TYPE`枚举指定，具体支持以下形式：
    - **CRYPT_PRIKEY_PKCS8_UNENCRYPT**：表示采用PKCS#8标准封装、未加密的私钥结构。
    - **CRYPT_PRIKEY_PKCS8_ENCRYPT**：表示采用PKCS#8标准封装并经过密码加密保护的私钥结构。
    - **CRYPT_PRIKEY_RSA**：表示RSA专用私钥结构。
    - **CRYPT_PRIKEY_ECC**：表示椭圆曲线算法的私钥ASN.1结构。
    - **CRYPT_PUBKEY_SUBKEY**：表示标准X.509`SubjectPublicKeyInfo`公钥封装结构。
    - **CRYPT_PUBKEY_RSA**：表示RSA算法专用公钥结构。
    - **CRYPT_PUBKEY_SUBKEY_WITHOUT_SEQ**：表示去除外层封装结构的原始公钥数据。

  - `encode`：输入参数，指向包含编码后ASN.1密钥数据的缓冲区。若格式为未知格式或PEM格式，缓冲区内容必须以`'\0'`结尾，且数据长度不应包含终止符。
  - `pwd`：输入参数，可选的密钥保护密码指针。若密钥未加密，可传入`NULL`。
  - `pwdlen`：输入参数，密码数据的长度，单位为字节。
  - `ealPKey`：输出参数，指向密钥上下文指针的地址。函数解析ASN.1缓冲区后，将创建并返回对应的密钥上下文结构。

- **返回值**：返回`HITLS_PKI_SUCCESS`。

#### **`CRYPT_EAL_DecodeFileKey`**

- **描述**：用于从指定格式的密钥文件中解析并解码非对称密钥数据，生成对应的密钥上下文结构，在后量子密码算法中常用于加载后量子公钥或私钥文件，以支持签名、验签或密钥封装等后续操作。

- **函数原型**： 

  ```c
  int32_t CRYPT_EAL_DecodeFileKey(int32_t format, int32_t type, const char *path, uint8_t *pwd, uint32_t pwdlen, CRYPT_EAL_PkeyCtx **ealPKey)
  ```

- **参数描述**：

  - `format`：输入参数，指定密钥文件的编码格式，当前支持传入枚举值`BSL_FORMAT_UNKNOWN`（自动判别格式）、`BSL_FORMAT_PEM`（PEM格式）或`BSL_FORMAT_ASN1`（DER格式）。
  - `type`：输入参数，指定密钥类型，表示待解码的非对称密钥类别，具体取值参考[CRYPT_EAL_DecodeBuffKey](#crypt_eal_decodebuffkey)中`type`参数的内容。
  - `path`：输入参数，密钥文件路径字符串，函数将从该路径读取密钥文件内容并进行解码。
  - `pwd`：输入参数，可选的密钥保护密码指针，用于解密受保护的私钥或公钥文件；若文件未加密，可传入`NULL`。
  - `pwdlen`：输入参数，密码数据的长度，单位为字节。
  - `ealPKey`：输出参数，指向密钥上下文指针的地址。函数解析密钥文件后，将创建并返回对应的密钥上下文结构。

- **返回值**：返回`HITLS_PKI_SUCCESS`。

#### **`CRYPT_EAL_EncodeBuffKey`**

- **描述**：用于将非对称密钥上下文中的密钥数据按照指定格式进行编码输出，在后量子密码算法中常用于导出后量子公钥或私钥，以便进行密钥存储、分发或证书生成。

- **函数原型**： 

  ```c
  int32_t CRYPT_EAL_EncodeBuffKey(CRYPT_EAL_PkeyCtx *ealPKey, const CRYPT_EncodeParam *encodeParam, int32_t format, int32_t type, BSL_Buffer *encode)
  ```

- **参数描述**：

  - `ealPKey`：输入参数，待编码的非对称密钥上下文结构指针，函数将从该上下文中提取密钥数据进行编码输出。
  - `encodeParam`：输入参数，PKCS8编码参数结构体，用于指定编码过程中使用的加密算法、密钥保护方式或相关编码配置。
  - `format`：输入参数，指定输出编码格式，当前支持传入枚举值`BSL_FORMAT_PEM`（PEM格式）或`BSL_FORMAT_ASN1`（DER格式）。
  - `type`：输入参数，指定密钥类型，表示待编码的非对称密钥类别，具体取值参考[CRYPT_EAL_DecodeBuffKey](#crypt_eal_decodebuffkey)中`type`参数的内容。
  - `encode`：输出参数，用于存放编码后的ASN.1格式密钥数据缓冲区，调用前需保证缓冲区具备足够的容量。

- **返回值**：返回`HITLS_PKI_SUCCESS`。

#### **`CRYPT_EAL_EncodeFileKey`**

- **描述**：用于将非对称密钥上下文中的密钥数据按照指定格式编码并保存到文件中，在后量子密码算法中常用于导出后量子公钥或私钥文件，以支持密钥存储、证书分发或系统间密钥交换。

- **函数原型**： 

  ```c
  int32_t CRYPT_EAL_EncodeFileKey(CRYPT_EAL_PkeyCtx *ealPKey, const CRYPT_EncodeParam *encodeParam, int32_t format, int32_t type, const char *path)
  ```

- **参数描述**：

  - `ealPKey`：输入参数，待编码的非对称密钥上下文结构指针，函数将从该上下文中提取密钥数据进行编码并写入指定文件。
  - `encodeParam`：输入参数，PKCS#8编码参数结构体，用于配置编码过程中的密钥保护方式、加密算法及相关编码选项。
  - `format`：输入参数，指定输出文件的密钥编码格式，当前支持传入枚举值`BSL_FORMAT_PEM`（PEM格式）或`BSL_FORMAT_ASN1`（DER格式）。
  - `type`：输入参数，指定密钥类型，表示待编码的非对称密钥类别，具体取值参考[CRYPT_EAL_DecodeBuffKey](#crypt_eal_decodebuffkey)中`type`参数的内容。
  - `path`：输入参数，目标密钥文件路径字符串，函数将编码后的密钥数据保存到该路径文件中。

- **返回值**：返回`HITLS_PKI_SUCCESS`。

### 密钥签名与验证

#### **`CRYPT_EAL_PkeySign`**

- **描述**：执行数字签名操作，使用指定的哈希算法和非对称密钥上下文对输入数据进行签名，在后量子密码算法中用于实现后量子签名方案的数据完整性验证和身份认证。

- **函数原型**： 

  ```c
  int32_t CRYPT_EAL_PkeySign(const CRYPT_EAL_PkeyCtx *pkey, CRYPT_MD_AlgId id, const uint8_t *data, uint32_t dataLen, uint8_t *sign, uint32_t *signLen)
  ```

- **参数描述**：

  - `pkey`：输入参数，非对称密钥会话结构指针，函数将基于该上下文中的私钥执行签名计算。
  - `id`：输入参数，指定用于签名计算的哈希算法标识符。
  - `data`：输入参数，待签名的原始数据缓冲区指针。
  - `dataLen`：输入参数，待签名数据的长度，取值范围为[0, 0xffffffff]。
  - `sign`：输出参数，用于存放生成的签名数据。调用者需保证缓冲区长度不小于密钥模数长度。
  - `signLen`：输入/输出参数，输入时表示签名缓冲区容量，输出时表示实际生成的签名长度。可通过 `CRYPT_EAL_PkeyGetSignLen` 获取推荐长度。

- **返回值**：返回`HITLS_PKI_SUCCESS`。

#### **`CRYPT_EAL_PkeyGetSignLen`**

- **描述**：用于获取当前密钥上下文对应签名算法的签名长度，在后量子签名算法中用于确定签名缓冲区大小，以确保签名数据能够完整存储。

- **函数原型**： 

  ```c
  uint32_t CRYPT_EAL_PkeyGetSignLen(const CRYPT_EAL_PkeyCtx *pkey)
  ```

- **参数描述**：

  - `pkey`：输入参数，非对称密钥会话结构指针，用于查询该密钥所对应签名算法的标准签名长度。

- **返回值**：返回当前密钥对应签名算法的签名长度，以字节为单位表示，失败返回0。

#### **`CRYPT_EAL_PkeyVerify`**

- **描述**：执行数字签名验证操作，使用指定的哈希算法和非对称密钥上下文对签名数据进行验证，在后量子密码算法中用于验证后量子签名方案生成的签名是否完整且可信。

- **函数原型**： 

  ```c
  int32_t CRYPT_EAL_PkeyVerify(const CRYPT_EAL_PkeyCtx *pkey, CRYPT_MD_AlgId id, const uint8_t *data, uint32_t dataLen, const uint8_t *sign, uint32_t signLen)
  ```

- **参数描述**：

  - `pkey`：输入参数，非对称密钥会话结构指针，函数将基于该上下文中的公钥执行签名验证操作。
  - `id`：输入参数，指定用于验证计算的哈希算法标识符。
  - `data`：输入参数，待验证签名的原始数据缓冲区指针。
  - `dataLen`：输入参数，待验证数据的长度，取值范围为[0, 0xffffffff]。
  - `sign`：输入参数，待验证的签名数据缓冲区指针。
  - `signLen`：输入参数，签名数据的长度。

- **返回值**：返回`HITLS_PKI_SUCCESS`。

## 证书相关接口

### 证书对象与编码

#### **`HITLS_X509_CertNew`**

- **描述**： 创建并初始化一个空的证书结构，分配并准备好证书内部的主体结构，为后续后量子算法信息的填充做好准备。

- **函数原型**： 

  ```c
  HITLS_X509_Cert *HITLS_X509_CertNew(void);
  ```

- **返回值**： 指向新创建的`HITLS_X509_Cert` 证书对象的指针，创建失败则返回`NULL`。

#### **`HITLS_X509_CertSign`**

- **描述**： 对证书的TBS部分进行合法性检查并使用指定私钥和签名算法生成数字签名，从而完成证书签发。当使用ML-DSA、SLH-DSA、XMSS等后量子签名算法时，私钥支持对应的后量子私钥。

- **函数原型**： 

  ```c
  int32_t HITLS_X509_CertSign(int32_t mdId, const CRYPT_EAL_PkeyCtx *prvKey, const HITLS_X509_SignAlgParam *algParam, HITLS_X509_Cert *cert)
  ```

- **参数描述**：

  - `mdId`：表示摘要算法标识，用于指定签名过程中使用的哈希算法
  - `prvkey`：指向私钥上下文结构的指针，用于执行签名操作。
  - `algParam`：指向签名算法参数结构的指针，用于指定签名算法的附加参数。
  - `cert`：指向待签名证书对象的指针。该对象中包含已构造完成的TBS结构。

- **返回值**： 返回`HITLS_PKI_SUCCESS`。

#### `HITLS_X509_CertCtrl` 

- **描述**： 用于处理证书的通用接口，通过传入不同的操作枚举值`cmd`和`void`类型指针`val`实现对证书的统一控制。在后量子证书场景中，可通过该接口配置包含后量子公钥标识、扩展字段及算法相关信息的证书结构内容，为后续签名与编码提供完整数据。

- **函数原型**： 

  ```c
  int32_t HITLS_X509_CertCtrl(HITLS_X509_Cert *cert, int32_t cmd, void *val, uint32_t valLen)
  ```

- **参数描述**：

  - `cert`：指向待操作的证书对象的指针，若为`NULL`则返回`HITLS_X509_ERR_INVALID_PARAM`表示参数错误。
  - `cmd`：控制命令标识，用于指定本次调用执行的具体操作类型，函数内部根据该命令进行分支调度。后量子证书中`HITLS_X509_SET_PUBKEY`和`HITLS_X509_GET_PUBKEY`将PQC公钥写入/读出证书，扩展命令控制中对ML-KEM、ML-DSA以及SLH-DSA证书的keyUsage限制尤为重要。
  - `val`：通用数据指针，用于传入或传出具体数据，函数内部根据`cmd`进行数据类型的转换。
  - `valLen`：表示`val`所指向数据的长度。

- **返回值**： 返回`HITLS_PKI_SUCCESS`。

#### **`HITLS_X509_CertParseBuff`** 

- **描述**： 该函数用于将输入缓冲区中的PEM或ASN.1编码证书数据解析为内部`HITLS_X509_Cert`结构对象，是后量子证书开发中实现后量子算法证书加载与验证流程的入口函数，把包含ML-DSA、SLH-DSA、XMSS等后量子签名算法标识及公钥信息的 X.509 证书二进制数据解析为可操作的内部结构，供后续进行算法识别、签名验证、公钥提取及扩展检查等操作。

- **函数原型**： 

  ```c
  int32_t HITLS_X509_CertParseBuff(int32_t format, const BSL_Buffer *encode, HITLS_X509_Cert **cert)
  ```

- **参数描述**：

  - `format`：指定证书编码格式，输入支持`BSL_FORMAT_PEM`（PEM 格式）、`BSL_FORMAT_ASN1`（DER/ASN.1二进制格式）和`BSL_FORMAT_UNKNOWN`（自动识别格式），当格式为PEM或UNKNOWN时，输入缓冲区必须以 `'\0'` 结尾。
  - `encode`：指向待解析证书数据的缓冲区结构指针，包含证书原始编码数据及长度信息；在PEM或UNKNOWN模式下，数据必须为以 `'\0'` 结尾的字符串形式。
  - `cert`：输出参数，用于接收解析成功后分配的证书对象指针；若解析成功，函数内部会申请内存并生成`HITLS_X509_Cert`实例，调用者在使用完毕后需要负责释放该对象。

- **返回值**： 返回`HITLS_PKI_SUCCESS`。

> 对应的Provider版本接口函数原型为：
> ```c
> int32_t HITLS_X509_ProviderCertParseBuff(HITLS_PKI_LibCtx *libCtx, const char *attrName, const char *format, const BSL_Buffer *encode, HITLS_X509_Cert **cert)
> ```
>
> 其中`libCtx`指向PKI库上下文对象的指针，用于指定当前使用的加密库环境，`attrName`指定Provider能力匹配的属性名称字符串，用于选择具体的Provider。
>
> 用于解析多个证书的接口和其Provider版本的函数原型分别为：
>
> ```c
> int32_t HITLS_X509_CertParseBundleBuff(int32_t format, const BSL_Buffer *encode, HITLS_X509_List **certlist);
> 
> int32_t HITLS_X509_ProviderCertParseBundleBuff(HITLS_PKI_LibCtx *libCtx, const char *attrName, const char *format, const BSL_Buffer *encode, HITLS_X509_List **certlist);
> ```
>
> 其中`certlist`用于接收解析成功后分配的多个证书组成的证书列表指针。

#### **`HITLS_X509_CertParseFile`**

- **描述**： 该函数用于从指定文件路径读取并解析PEM或ASN.1格式的X.509证书数据，生成内部`HITLS_X509_Cert`对象，是后量子证书开发中实现后量子证书加载与验证流程的文件入口接口，负责将磁盘中存储的包含ML-DSA、SLH-DSA、XMSS等后量子签名算法标识及公钥信息的证书文件解析为内部证书结构，供后续进行公钥提取、签名验证、扩展检查及算法识别等操作。

- **函数原型**： 

  ```c
  int32_t HITLS_X509_CertParseFile(int32_t format, const char *path, HITLS_X509_Cert **cert)
  ```

- **参数描述**：

  - `format`：指定证书编码格式，输入支持`BSL_FORMAT_PEM`（PEM 格式）、`BSL_FORMAT_ASN1`（DER/ASN.1 二进制格式）和`BSL_FORMAT_UNKNOWN`（自动识别格式）。
  - `path`：指向证书文件路径的字符串指针；函数根据该路径读取证书文件内容并进行解析。
  - `cert`：输出参数，用于接收解析成功后分配的证书对象指针；若解析成功，函数内部会申请内存并生成`HITLS_X509_Cert`实例，调用者在使用完毕后需要负责释放该对象。

- **返回值**： 返回`HITLS_PKI_SUCCESS`。

> 对应的Provider版本接口函数原型为：
>
> ```c
> int32_t HITLS_X509_ProviderCertParseFile(HITLS_PKI_LibCtx *libCtx, const char *attrName, const char *format, const char *path, HITLS_X509_Cert **cert)
> ```
>
> 其中`libCtx`指向PKI库上下文对象的指针，用于指定当前使用的加密库环境，`attrName`指定 Provider 能力匹配的属性名称字符串，用于选择具体的Provider。
>
> 用于解析多个证书的接口和其Provider版本的函数原型分别为：
>
> ```c
> int32_t HITLS_X509_CertParseBundleFile(int32_t format, const char *path, HITLS_X509_List **certlist)
> 
> int32_t HITLS_X509_ProviderCertParseBundleFile(HITLS_PKI_LibCtx *libCtx, const char *attrName, const char *format, const char *path, HITLS_X509_List **certlist)
> ```
>
> 其中`certlist`用于接收解析成功后分配的多个证书组成的证书列表指针。

> 同样地，CRL、CSR和PKCS#12在生成、解析和编码的接口和参数上与上述证书结构`HITLS_X509_Cert`基本保持一致。

#### **`HITLS_X509_CertGenBuff`**

- **描述**： 该函数用于将内部 `HITLS_X509_Cert` 证书对象编码生成 PEM 或 ASN.1 格式的二进制证书数据，是后量子证书开发中实现证书存储、传输与发布的重要输出接口，负责将包含后量子签名算法（如 ML-DSA、SLH-DSA、XMSS 等）及相关扩展字段的证书结构序列化为标准编码格式，便于证书分发、持久化存储以及外部系统验证。

- **函数原型**： 

  ```c
  int32_t HITLS_X509_CertParseFile(int32_t format, const char *path, HITLS_X509_Cert **cert)
  ```

- **参数描述**：

  - `format`：指定证书编码格式，输入支持`BSL_FORMAT_PEM`（PEM 格式）和`BSL_FORMAT_ASN1`（DER/ASN.1 二进制格式）。
  - `cert`：指向待编码的证书对象指针，函数将读取该证书结构中的 TBS 信息、签名信息以及扩展字段，并进行标准编码输出。
  - `buff`：输出缓冲区指针，用于存储编码后的证书数据。

- **返回值**： 返回`HITLS_PKI_SUCCESS`。

> 与之类似的将证书编码到指定路径的`HITLS_X509_CertGenFile`接口函数原型为：
>
> ```c
> int32_t HITLS_X509_CertGenFile(int32_t format, HITLS_X509_Cert *cert, const char *path)
> ```
>
> 区别在于`path`作为输出文件路径的字符串指针，字符串表示的路径存储编码后的证书数据。

### 证书链构建与验证

#### **`HITLS_X509_CertNew`**

- **描述**： 创建并初始化证书存储上下文`HITLS_X509_StoreCtx`对象，使系统能够加载和管理支持后量子签名算法的可信证书，从而为后量子证书链验证提供运行环境。

- **函数原型**： 

  ```c
  HITLS_X509_StoreCtx *HITLS_X509_StoreCtxNew(void)
  ```

- **返回值**： 指向新创建的`HITLS_X509_StoreCtx` 证书存储上下文对象的指针，创建失败则返回`NULL`。

#### `HITLS_X509_StoreCtxCtrl` 

- **描述**： 通过控制命令方式对证书存储上下文 `StoreCtx` 进行参数配置、证书链管理、验证回调设置及状态查询等操作，是后量子证书链验证环境配置与运行控制的核心接口。

- **函数原型**： 

  ```c
  int32_t HITLS_X509_StoreCtxCtrl(HITLS_X509_StoreCtx *storeCtx, int32_t cmd, void *val, uint32_t valLen)
  ```

- **参数描述**：

  - `storeCtx`：指向证书存储上下文对象的指针，是函数控制操作的目标对象，该上下文主要用于管理证书链验证策略及可信证书集合。
  - `cmd`：控制命令标识，用于指定本次操作类型。支持多种控制指令，函数内部根据命令值进行分支调度处理。
  - `val`：通用数据指针，用于传入或传出具体数据，函数内部根据`cmd`进行数据类型的转换。
  - `valLen`：表示`val`所指向数据的长度。

- **返回值**： 返回`HITLS_PKI_SUCCESS`。

#### `HITLS_X509_CertVerify` 

- **描述**： 该函数用于基于指定证书存储上下文对输入的证书链进行合法性验证，通过使用存储上下文中配置的验证策略和可信证书集合，对包含后量子签名算法（如 ML-DSA、SLH-DSA、XMSS 等）的证书链进行逐级验证，确保证书签名有效性、链结构正确性以及算法兼容性，从而支撑后量子安全通信认证流程。

- **函数原型**： 

  ```c
  int32_t HITLS_X509_CertVerify(HITLS_X509_StoreCtx *storeCtx, HITLS_X509_List *chain)
  ```

- **参数描述**：

  - `storeCtx`：指向证书存储上下文对象的指针，函数将依据该上下文中配置的可信证书库、验证策略及安全参数执行证书链验证流程。
  - `chain`：指向待验证的证书链列表结构指针，链表中通常包含终端实体证书及其上级CA证书，函数将按证书链顺序执行签名验证及结构合法性检查。

- **返回值**： 返回`HITLS_PKI_SUCCESS`。

#### `HITLS_X509_CertVerifyByPubKey` 

- **描述**： 该函数用于使用外部提供的公钥上下文对单个证书的数字签名进行验证，支持使用ML-DSA、SLH-DSA、XMSS等后量子签名算法对应的公钥对证书签名值进行验证，适用于不依赖证书链验证的场景。

- **函数原型**： 

  ```c
  int32_t HITLS_X509_CertVerifyByPubKey(HITLS_X509_Cert *cert, CRYPT_EAL_PkeyCtx *pubKey)
  ```

- **参数描述**：

  - `cert`：指向待验证签名的证书对象指针，函数将提取证书中的签名数据及证书编码内容进行验证，调用前证书必须已完成解析或生成。
  - `pubKey`：指向公钥上下文结构的指针，用于提供验证证书签名所需的公钥信息。

- **返回值**： 返回`HITLS_PKI_SUCCESS`。

#### `HITLS_X509_CheckKey` 

- **描述**： 该函数通过签名验证方式检查证书中的公钥与给定私钥是否匹配，可用于验证ML-DSA、SLH-DSA、XMSS等后量子签名算法的公私钥一致性，通常用于证书生成前的密钥正确性检查，防止错误密钥被用于证书签发。

- **函数原型**： 

  ```c
  int32_t HITLS_X509_CheckKey(HITLS_X509_Cert *cert, CRYPT_EAL_PkeyCtx *prvKey)
  ```

- **参数描述**：

  - `cert`：指向包含待校验公钥的证书对象指针；函数通过对证书数据执行签名验证操作来判断公钥与私钥是否属于同一密钥对。
  - `prvKey`：指向私钥上下文结构指针，用于提供参与匹配验证的私钥信息；在后量子证书场景中，该私钥对应后量子签名算法的私钥上下文。

- **返回值**： 返回`HITLS_PKI_SUCCESS`。

### CMS接口

#### **`HITLS_CMS_ProviderNew`**

- **描述**：基于PKI库上下文与 Provider 属性创建一个新的CMS（Cryptographic Message Syntax）处理句柄，可以用于创建支持后量子算法（如ML-DSA、SLH-DSA等）的CMS处理上下文，使系统能够生成或解析包含后量子签名的签名数据，用于证书分发、签名封装及安全消息交换。

- **函数原型**： 

  ```c
  HITLS_CMS *HITLS_CMS_ProviderNew(HITLS_PKI_LibCtx *libCtx, const char *attrName, int32_t dataType)
  ```

- **参数描述**： 

  - `libCtx`：指向 PKI 库上下文对象的指针，用于指定当前算法运行环境，在Provider机制下，该上下文用于查找并加载支持特定算法的实现模块。
  - `attrName`：Provider属性或配置名称字符串，用于选择具备特定能力的CMS实现模块。
  - `dataType`：指定CMS内容类型用于确定CMS句柄后续处理的数据结构类型，当前接口仅支持PKCS#7结构中的签名数据结构SignedData，枚举值表示为`BSL_CID_PKCS7_SIGNEDDATA`。

- **返回值**： 指向新创建的`HITLS_CMS`证书对象的指针，创建失败则返回`NULL`。

#### **`HITLS_CMS_DataSign`**

- **描述**：该函数用于在 CMS签名数据结构SignedData中创建签名者信息结构SignerInfo并根据需要执行一次性数据签名操作，可使用ML-DSA等后量子签名私钥对消息进行签名，并将签名结果与签名者证书封装进CMS的SignedData结构中，从而实现符合CMS/PKCS#7标准的后量子签名数据发布与交换。

- **函数原型**： 

  ```c
  int32_t HITLS_CMS_DataSign(HITLS_CMS *cms, CRYPT_EAL_PkeyCtx *prvKey, HITLS_X509_Cert *cert, BSL_Buffer *msg, const BSL_Param *optionalParam);
  ```

- **参数描述**： 

  - `cms`：指向CMS句柄的指针，必须为SignedData类型，函数将在该CMS结构中创建并加入SignerInfo结构。

  - `prvKey`：指向签名私钥上下文的指针，在执行一次性签名的`msg`参数非`NULL`的情况下为必需参数，在仅创建SignerInfo的流式签名模式时可为`NULL`。后量子算法中支持ML-DSA的算法私钥，后量子算法签名需要从`optionParam`参数中获得摘要算法信息和是否存在签名属性集合SignedAttrs，以满足RFC 9882中的规范：

    - 若不存在SignedAttrs，使用ML-DSA算法签名必须使用SHA-512作为摘要算法以最大程度地减少交互故障；
    - 若存在SignedAttrs，ML-DSA算法需满足如下的摘要算法对应关系：

    | ML-DSA算法 | 可用的摘要算法                                               |
    | ---------- | ------------------------------------------------------------ |
    | ML-DSA-44  | SHA256、SHA384、SHA512、SHA3-256、SHA3-384、SHA3-512、SHAKE128、SHAKE256 |
    | ML-DSA-65  | SHA384、SHA512、SHA3-384、SHA3-512、SHAKE256                 |
    | ML-DSA-87  | SHA512、SHA3-512、SHAKE256                                   |

    > 如果ML-DSA算法和摘要算法不满足上述对应关系，则返回错误`HITLS_CMS_ERR_MLDSA_INVALID_DIGEST`表示摘要无效

  - `cert`：签名者证书，用于构造 SignerInfo 中的签名者标识信息，例如issuer和serialNumber等信息，并可嵌入CMS结构中。

  - `msg`：表示待签名的消息数据缓冲区，如果不为`NULL`，则执行一次性签名流程并生成签名值；如果为`NULL`，则仅创建SignerInfo，后续可通过流式接口完成签名。

  - `optionalParam`：可选参数结构指针，包含额外的未受信证书列表、CA证书列表或其他扩展配置，可以被设置为`NULL`。

- **返回值**： 返回`HITLS_PKI_SUCCESS`。

#### **`HITLS_CMS_DataVerify`**

- **描述**：该函数用于验证CMS的SignedData 结构中的全部签名，可对使用ML-DSA等后量子签名算法生成的 CMS的SignedData进行签名验证，结合证书链与可选信任参数判断签名是否有效，从而保障后量子安全消息交换的真实性与完整性。

- **函数原型**： 

  ```c
  int32_t HITLS_CMS_DataVerify(HITLS_CMS *cms, BSL_Buffer *msg, const BSL_Param *inputParam, BSL_Buffer *output)
  ```

- **参数描述**： 

  - `cms`：指向包含待验证签名的CMS结构体指针；必须为SignedData类型，函数将遍历其中的所有SignerInfo进行签名验证。对于后量子算法签名的验证，该函数对`cms`结构的SignedData进行验证：
    - 若不存在SignedAttrs，摘要算法必须为SHA512，否则返回错误`HITLS_CMS_ERR_MLDSA_ERROR_DIGEST`。
    - 若存在SignedAttrs，ML-DSA算法必须与摘要算法满足[签名部分](#hitls_cms_datasign)的对应关系，否则返回错误`HITLS_CMS_ERR_MLDSA_INVALID_DIGEST`。

  - `msg`：指向待验证消息数据的缓冲区，在分离式签名的情况下不能为`NULL`，在非分离签名的情况下可以为`NULL`，此时验证使用CMS内嵌内容。
  - `inputParam`：可选输入参数结构指针，包含额外的未受信证书列表、CA证书列表或其他验证辅助信息，可以被设置为`NULL`。
  - `output`：输出参数，用于返回实际参与验证的消息数据。在分离式签名的情况下指向`msg`，在非分离签名的情况下指向CMS内嵌的内容数据，如果不需要返回消息数据可以设置为`NULL`。

- **返回值**： 返回`HITLS_PKI_SUCCESS`。

## ML-KEM使用指南

### 初始设置
后量子证书开发过程中首先需要完成初始化工作，具体包括错误码和随机数的初始化以及malloc和free函数的注册。

```c
// 初始化错误码模块
BSL_ERR_Init();

// 调用算法API接口之前需要调用BSL_SAL_CallBack_Ctrl函数注册malloc和free函数。该步骤仅需执行一次
// 如果未注册并且默认能力没有被裁剪，使用默认linux实现，StdMalloc为自定义的malloc函数作为演示
BSL_SAL_CallBack_Ctrl(BSL_SAL_MEM_MALLOC, StdMalloc);
BSL_SAL_CallBack_Ctrl(BSL_SAL_MEM_FREE, free);

// 初始化随机数，此处以SHA256算法为例，否则会报错CRYPT_NO_REGIST_RAND
ret = CRYPT_EAL_ProviderRandInitCtx(NULL, CRYPT_RAND_SHA256, "provider=default", NULL, 0, NULL);
if (ret != CRYPT_SUCCESS) {
    printf("CRYPT_EAL_PkeyDecrypt: error code is %x\n", ret);
    goto EXIT;
}
```

> 为了保证伪代码可读性，后续伪代码省略返回值不符合预期时的处理逻辑。

### 密钥设置

1. **创建ML-KEM密钥对**

   ```c
   // 根据ML-KEM对应的算法ID创建非对称密钥上下文
   CRYPT_EAL_PkeyCtx *ctx = CRYPT_EAL_PkeyNewCtx(CRYPT_PKEY_ML_KEM);
   // 根据参数id设置参数，参数id可选 CRYPT_KEM_TYPE_MLKEM_512/ CRYPT_KEM_TYPE_MLKEM_768/ CRYPT_KEM_TYPE_MLKEM_1024
   uint32_t id = CRYPT_KEM_TYPE_MLKEM_512;
   int32_t ret = CRYPT_EAL_PkeySetParaById(ctx, id);
   // 初始化随机数
   ret = CRYPT_EAL_ProviderRandInitCtx(NULL, CRYPT_RAND_SHA256, "provider=default", NULL, 0, NULL);
   // 生成密钥对保存在上下文中
   ret = CRYPT_EAL_PkeyGen(ctx);
   ```

   **典型错误枚举**：

   | 枚举值                           | 错误原因               | 触发场景                                      |
   | -------------------------------- | ---------------------- | --------------------------------------------- |
   | `CRYPT_MLKEM_CTRL_INIT_REPEATED` | 密钥上下文被多次初始化 | 调用多次`CRYPT_EAL_PkeySetParaById`函数       |
   | `CRYPT_MLKEM_KEYINFO_NOT_SET`    | 未设置算法类型         | 未调用`CRYPT_EAL_PkeySetParaById`直接执行操作 |

2. **密钥控制**

   ```c
   // 设置私钥编码格式
   uint32_t dkFormat = CRYPT_ALGO_MLKEM_DK_FORMAT_SEED_ONLY;
   int32_t ret = CRYPT_EAL_PkeyCtrl(pkey, CRYPT_CTRL_SET_MLKEM_DK_FORMAT, &dkFormat, sizeof(dkFormat));
                         
   // 获取私钥编码格式
   uint32_t curDkFmt = 0;
   ret = CRYPT_EAL_PkeyCtrl(ealPriKey, CRYPT_CTRL_GET_MLKEM_DK_FORMAT, &curDkFmt, sizeof(curDkFmt));
                                    
   // 获取私钥种子
   uint8_t seed[64] = {0};
   ret = CRYPT_EAL_PkeyCtrl(ealPriKey, CRYPT_CTRL_GET_MLKEM_SEED, seed, MLKEM_SEED_BYTES_LEN);
   ```

   **典型错误枚举**：

   | 枚举值                         | 错误原因                       | 触发场景                                                     |
   | ------------------------------ | ------------------------------ | ------------------------------------------------------------ |
   | `CRYPT_MLKEM_CTRL_NOT_SUPPORT` | MLKEM算法不支持该操作码        | `CRYPT_EAL_PkeyCtrl`函数`opt`参数传入异常值                  |
   | `CRYPT_MLKEM_SEED_NOT_SET`     | 尝试读取种子但上下文中并未记录 | 对仅保存私钥数据的密钥上下文调用`CRYPT_EAL_PkeyCtrl(pkey, CRYPT_CTRL_GET_MLKEM_SEED, ...)` |

3. **密钥对的编解码**

   **密钥编码**

   ```c
   // 按照创建密钥对的方式得到非对称密钥上下文pkey后用来编码
   
   // 1. 编码到缓冲区
   // 初始化缓冲区
   BSL_Buffer out = {0};
   // 设置私钥编码格式
   uint32_t dkFormat = CRYPT_ALGO_MLKEM_DK_FORMAT_SEED_ONLY;
   int32_t ret = CRYPT_EAL_PkeyCtrl(pkey, CRYPT_CTRL_SET_MLKEM_DK_FORMAT, &dkFormat, sizeof(dkFormat));
   // 调用编码接口，设置format和type参数，得到存放编码的缓冲区out
   ret = CRYPT_EAL_EncodeBuffKey(pkey, NULL, BSL_FORMAT_PEM, CRYPT_PRIKEY_PKCS8_UNENCRYPT, &out);
   
   // 2. 编码到文件
   int32_t format = BSL_FORMAT_PEM;
   int32_t type = CRYPT_PRIKEY_PKCS8_UNENCRYPT;
   // 设置私钥编码格式
   uint32_t dkFormat = CRYPT_ALGO_MLKEM_DK_FORMAT_SEED_ONLY;
   int32_t ret = CRYPT_EAL_PkeyCtrl(pkey, CRYPT_CTRL_SET_MLKEM_DK_FORMAT, &dkFormat, sizeof(dkFormat));
   // path参数指明编码密钥的文件路径
   ret = CRYPT_EAL_EncodeFileKey(pkey, NULL, format, type, "mlkem512_gen_seed.pem");
   ```

   ML-KEM密钥对编码前必须完成私钥编码表示形式的设置，具体通过调用`CRYPT_EAL_PkeyCtrl`接口并传入`CRYPT_CTRL_SET_MLKEM_DK_FORMAT`命令以及设置的编码格式枚举。编码格式由`CRYPT_ALGO_MLKEM_DK_FORMAT_TYPE`枚举指定，共支持以下三种形式：

   + **CRYPT_ALGO_MLKEM_DK_FORMAT_BOTH**：包括生成私钥所需种子和展开后私钥数据的完整格式
   + **CRYPT_ALGO_MLKEM_DK_FORMAT_DK_ONLY**：仅包含展开后私钥数据的格式
   + **CRYPT_ALGO_MLKEM_DK_FORMAT_SEED_ONLY**：仅包含生成私钥所需种子的格式

   **密钥解码**

   ```c
   // 1. 从缓冲区解码密钥对
   int32_t ret = CRYPT_EAL_PkeyCtx *prvKey = NULL;
   // 设置format和type，keyBuf为存储密钥对编码的缓冲区
   ret = CRYPT_EAL_DecodeBuffKey(BSL_FORMAT_PEM, CRYPT_PRIKEY_PKCS8_UNENCRYPT, &keyBuf, NULL, 0, &prvKey);
   
   // 2. 从文件解码密钥对
   int32_t ret = CRYPT_EAL_PkeyCtx *prvKey = NULL;
   ret = CRYPT_EAL_DecodeBuffKey(BSL_FORMAT_PEM, CRYPT_PRIKEY_PKCS8_UNENCRYPT, "mlkem512_prikey_seed.pem", NULL, 0, &prvKey);
   ```

   **设置公私钥**

   ```c
   // 设置公钥
   CRYPT_EAL_PkeyPub ek = { 0 };
   // 设置算法类型
   ek.id = CRYPT_PKEY_ML_KEM;
   // 开辟空间并复制公钥数据
   ek.key.kemEk.data =  BSL_SAL_Malloc(encapsKeyLen);
   (void)memcpy_s(ek.key.kemEk.data, encapsKeyLen, pubkey->x, pubkey->len);
   ek.key.kemEk.len = encapsKeyLen;
   // 调用函数将公钥结构存放如入上下文
   ret = CRYPT_EAL_PkeySetPub(ctx, &ek);
   
   // 设置私钥
   CRYPT_EAL_PkeyPrv dk = { 0 };
   // 设置算法类型
   dk.id = CRYPT_PKEY_ML_KEM;
   // 开辟空间并复制私钥数据
   dk.key.kemDk.data =  BSL_SAL_Malloc(decapsKeyLen);
   (void)memcpy_s(dk.key.kemDk.data, decapsKeyLen, prvkey->x, prvkey->len);
   dk.key.kemDk.len = decapsKeyLen;
   // 调用函数将私钥结构存放如入上下文
   ret = CRYPT_EAL_PkeySetPrv(ctx, &dk);
   ```

   **典型错误枚举**：

   |枚举值|错误原因| 触发场景|
   | -------------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
   | `CRYPT_MLKEM_KEY_NOT_SET`                    | 密钥上下文未填充公钥或私钥进行获取                           | 在上下文公钥或私钥不存在的情况下调用`CRYPT_EAL_PkeyGetPrv`或`CRYPT_EAL_PkeyGetPub`函数 |
   | `CRYPT_MLKEM_KEY_REPEATED_SET`               | 密钥上下文已经填充公钥或私钥                                 | 在上下文公钥或私钥存在的情况下调用`CRYPT_EAL_PkeySetPrv`或`CRYPT_EAL_PkeySetPub`函数 |
   | `CRYPT_MLKEM_DECODE_KEY_OVERFLOW`            | 解码公私钥的值越界                                           | `CRYPT_EAL_PkeySetPub/SetPrv`传入畸形密钥导致内部解码失败    |
   | `CRYPT_MLKEM_SEED_EXPANDED_KEY_INCONSISTENT` | 种子生成的密钥与提供的扩展密钥不匹配或私钥解码得到的$z$值不匹配 | `CRYPT_EAL_DecodeFileKey`的`key`参数或调用`CRYPT_EAL_PkeySetPrv`函数传入的`pkey`参数与种子生成的密钥不一致 |
   | `CRYPT_MLKEM_INVALID_PRVKEY`                 | 私钥为空或长度错误或成对一致性检查失败                       | 调用`CRYPT_EAL_DecodeFileKey`传入私钥为空/私钥长度错误/私钥结构被破坏的参数或开启`HITLS_CRYPTO_MLKEM_CHECK`开关后调用`CRYPT_EAL_PkeyPrvCheck`触发 |
   | `CRYPT_MLKEM_PAIRWISE_CHECK_FAIL`            | 公私钥不匹配                                                 | 开启`HITLS_CRYPTO_MLKEM_CHECK`开关后调用`CRYPT_EAL_PkeyPairCheck`触发 |
   | `CRYPT_MLKEM_DK_FORMAT_ERROR`                | 私钥编码格式设置错误或要求种子的格式种子缺失                 | 调用`CRYPT_EAL_PkeyCtrl(pkey, CRYPT_CTRL_SET_MLKEM_DK_FORMAT, ...)`函数设置编码格式后调用`CRYPT_EAL_EncodeBuffKey/EncodeFileKey`导出私钥 |

4. **密钥封装**

   ```c
   // 初始化密钥封装上下文
   ret = CRYPT_EAL_PkeyEncapsInit(pubkey_ctx, NULL);
   // 执行密钥封装
   ret = CRYPT_EAL_PkeyEncaps(pubkey_ctx, ciphertext, &cipherLen, sharedKey, &sharedLen);
   
   // 将解封装输出长度初始化为共享密钥缓冲区容量
   decSharedLen = sharedLen;
   
   // 初始化密钥解封装上下文
   ret = CRYPT_EAL_PkeyDecapsInit(prvkey_ctx, NULL);
   ret = CRYPT_EAL_PkeyDecaps(prvkey_ctx, ciphertext, cipherLen, decSharedKey, &decSharedLen);
   ```

   **典型错误枚举**：

   | 枚举值                       | 错误原因                           | 触发场景                                                     |
   | ---------------------------- | ---------------------------------- | ------------------------------------------------------------ |
   | `CRYPT_MLKEM_LEN_NOT_ENOUGH` | 密钥封装或解封装输出缓冲区空间不足 | 调用`CRYPT_EAL_PkeyEncaps`或`CRYPT_EAL_PkeyDecaps`传入的`cipher`或`shareKey`长度不足 |
   | `CRYPT_MLKEM_INVALID_PRVKEY` | 解封装对私钥的有效性检查失败       | 调用`CRYPT_EAL_PkeyDecaps`函数传入无效私钥结构触发           |


### 证书设置

1. **证书创建和解析**

   ```c
   // 创建一个新证书
   HITLS_X509_Cert *cert = HITLS_X509_CertNew();
   if (cert == NULL) {
       printf("创建空证书对象失败\n");
       return HITLS_X509_ERR_CERT_EXIST;
   }
   
   // 从文件中读取证书
   HITLS_X509_Cert *cert = NULL;
   HITLS_X509_CertParseFile(BSL_FORMAT_PEM, "mlkem_end.crt", &cert);
   if (cert == NULL) {
       printf("创建空证书对象失败\n");
       return HITLS_X509_ERR_CERT_EXIST;
   }
   ```

2. **证书控制**

   ```c
   // 设置证书相关信息并配置公钥
   ret = HITLS_X509_CertCtrl(cert, HITLS_X509_SET_VERSION, &version, sizeof(version));
   ret = HITLS_X509_CertCtrl(cert, HITLS_X509_SET_SERIALNUM, serialNum, sizeof(serialNum));
   ···
   ret = HITLS_X509_CertCtrl(ert, HITLS_X509_SET_PUBKEY, pubkey, 0);
   
   // 定义KeyUsage 扩展：ML-KEM证书仅允许keyEncipherment以符合draft-ietf-lamps-kyber-certificates要求
   HITLS_X509_ExtKeyUsage ku;
   ku.critical = true;
   ku.keyUsage = HITLS_X509_EXT_KU_KEY_ENCIPHERMENT;
   ret = HITLS_X509_CertCtrl(cert, HITLS_X509_EXT_SET_KUSAGE, &ku, sizeof(HITLS_X509_ExtKeyUsage));
   ```
   
   > `KeyUsage`参数在ML-KEM算法中作为可选项，如要设置仅允许设置为`HITLS_X509_EXT_KU_KEY_ENCIPHERMENT`，表示用于密钥封装，否则不满足draft-ietf-lamps-kyber-certificates要求在后续验证环节失败；根据RFC 5280，`KeyUsage`如果不存在则不适用任何限制，相应地在验证环节也应验证成功。
   
3. **证书签名与验证**

   ```c
   // 签发ML-KEM证书
   ret = HITLS_X509_CertSign(CRYPT_MD_SHA256, signKey, &algParam, cert);

   ret = HITLS_X509_CertGenFile(BSL_FORMAT_PEM, cert, "mlkem_ee.pem");
   
   // 1. 使用公钥进行验证
   ret = HITLS_X509_CertVerifyByPubKey(cert, pubkey);
   
   // 2. 构建证书链进行验证
   // 定义存储上下文和证书链指针
   HITLS_X509_StoreCtx *store = HITLS_X509_StoreCtxNew();
   HITLS_X509_List *chain = NULL;
   
   // 加入中间CA证书到上下文
   HITLS_X509_Cert *inter = NULL;
   ret = HITLS_X509_CertParseFile(BSL_FORMAT_UNKNOWN, "mlkem_inter.pem", &inter);
   
   ret = HITLS_X509_StoreCtxCtrl(store, HITLS_X509_STORECTX_DEEP_COPY_SET_CA, inter, sizeof(HITLS_X509_Cert));
   
   // 解析终端证书
   HITLS_X509_Cert *entity = NULL;
   ret = HITLS_X509_CertParseFile(BSL_FORMAT_PEM, "mlkem_entity.pem", &entity);
   
   // 构建证书链
   ret = HITLS_X509_CertChainBuild(store, false, entity, &chain);
   
   // 将根证书加入上下文
   HITLS_X509_Cert *root = NULL;
   ret = HITLS_X509_CertParseFile(BSL_FORMAT_PEM, "mlkem_root.pem", &root);
   
   ret = HITLS_X509_StoreCtxCtrl(store, HITLS_X509_STORECTX_DEEP_COPY_SET_CA, root, sizeof(HITLS_X509_Cert));
   
   // 重新构建证书链
   ret = HITLS_X509_CertChainBuild(store, false, entity, &chain);
   
   // 验证证书链
   ret = HITLS_X509_CertVerify(store, chain);
   ```
   
   **典型错误枚举**：
   
   | 枚举值                  | 错误原因                                                   | 触发场景                                                     |
   | ----------------------- | ---------------------------------------------------------- | ------------------------------------------------------------ |
   | `HITLS_X509_ERR_EXT_KU` | 证书设置`HITLS_X509_ExtKeyUsage`结构设置`keyUsage`参数错误 | `HITLS_X509_CertCtrl`函数传入的`HITLS_X509_ExtKeyUsage`结构`keyUsage`不为`keyEncipherment` |

### 密钥设置

1. **创建ML-DSA密钥对**

   ```c
   // 根据ML-DSA对应的算法ID创建非对称密钥上下文
   CRYPT_EAL_PkeyCtx *ctx = CRYPT_EAL_PkeyNewCtx(CRYPT_PKEY_ML_DSA);
   // 根据参数id设置参数，参数id可选 CRYPT_MLDSA_TYPE_MLDSA_44/ CRYPT_MLDSA_TYPE_MLDSA_65/ CRYPT_MLDSA_TYPE_MLDSA_87
   uint32_t id = CRYPT_MLDSA_TYPE_MLDSA_65;

   // 初始化随机数
   ret = CRYPT_EAL_ProviderRandInitCtx(NULL, CRYPT_RAND_SHA256, "provider=default", NULL, 0, NULL);
   ```

   **典型错误枚举**：

   | 枚举值                           | 错误原因                       | 触发场景                                                     |
   | -------------------------------- | ------------------------------ | ------------------------------------------------------------ |
   | `CRYPT_MLDSA_KEYINFO_NOT_SET`    | 未设置算法参数                 | 未调用`CRYPT_EAL_PkeySetParaById`函数设置算法                |
   | `CRYPT_MLDSA_CTRL_INIT_REPEATED` | 密钥上下文被多次初始化         | 调用多次`CRYPT_EAL_PkeySetParaById`函数                      |
   | `CRYPT_MLDSA_CTRL_NOT_SUPPORT`   | MLDSA算法不支持该操作码        | `CRYPT_EAL_PkeyCtrl`函数`opt`参数传入异常值                  |
   | `CRYPT_MLDSA_SEED_NOT_SET`       | 尝试读取种子但上下文中并未记录 | 对仅保存私钥数据的密钥上下文调用`CRYPT_EAL_PkeyCtrl(pkey, CRYPT_CTRL_GET_MLDSA_SEED, ...)` |

2. **密钥控制**

   ```c
   // 设置私钥编码格式
   uint32_t dkFormat = CRYPT_ALGO_MLDSA_PRIV_FORMAT_PRIV_ONLY;
   int32_t ret = CRYPT_EAL_PkeyCtrl(pkey, CRYPT_CTRL_SET_MLDSA_PRVKEY_FORMAT, &dkFormat, sizeof(dkFormat));
                                    
   // 设置确定性签名模式，0表示随机签名，1表示确定性签名
   uint32_t deterministic = 1;
   CRYPT_EAL_PkeyCtrl(ctx, CRYPT_CTRL_SET_DETERMINISTIC_FLAG, &deterministic, sizeof(deterministic));
                                    
   // 设置ML-DSA的上下文字段 
   const uint8_t ctx[] = "FDE19259E56C2602F3CB0DA509B912F88262A1701D4E02B513F45C97EBB100A";
   uint32_t ctx_len = (uint32_t)(sizeof(ctx) - 1U);
   ret = CRYPT_EAL_PkeyCtrl(pkey, CRYPT_CTRL_SET_CTX_INFO, &ctx, ctx_len);
                                    
   // 控制签名时是否使用预哈希版本，0表示不使用预哈希版本，1表示使用预哈希版本 
   int32_t prehash = 0;
   ret = CRYPT_EAL_PkeyCtrl(pkey, CRYPT_CTRL_SET_PREHASH_MODE, &prehash, sizeof(prehash));
                                    
   // 控制是否使用外部的μ参数，0表示库内部计算参数μ，1表示μ来自外部提供 
   int32_t externalMu = 0; 
   ret = CRYPT_EAL_PkeyCtrl(pkey, CRYPT_CTRL_SET_MLDSA_MUMSG_FLAG, &externalMu, sizeof(externalMu));
                                    
   // 获取当前私钥编码格式
   uint32_t curFmt = 0;
   CRYPT_EAL_PkeyCtrl(pctx, CRYPT_CTRL_GET_MLDSA_PRVKEY_FORMAT, &curFmt, sizeof(curFmt));
                                    
   // 获取私钥种子
   uint8_t seed[MLDSA_SEED_BYTES_LEN] = {0};
   int32_t ret = CRYPT_EAL_PkeyCtrl(pctx, CRYPT_CTRL_GET_MLDSA_SEED, seed, sizeof(seed));
   ```

3. **密钥对的编解码**

   **密钥编码**

   ```c
   // 按照创建密钥对的方式得到非对称密钥上下文pkey后用来编码
   
   // 1. 编码到缓冲区
   // 初始化缓冲区
   BSL_Buffer out = {0};
   // 设置私钥编码格式
   uint32_t dkFormat = CRYPT_ALGO_MLDSA_PRIV_FORMAT_PRIV_ONLY;
   int32_t ret = CRYPT_EAL_PkeyCtrl(pkey, CRYPT_CTRL_SET_MLDSA_PRVKEY_FORMAT, &dkFormat, sizeof(dkFormat));
   // 调用编码接口，设置format和type参数，得到存放编码的缓冲区out
   ret = CRYPT_EAL_EncodeBuffKey(pkey, NULL, BSL_FORMAT_PEM, CRYPT_PRIKEY_PKCS8_UNENCRYPT, &out);
   
   // 2. 编码到文件
   int32_t format = BSL_FORMAT_PEM;
   int32_t type = CRYPT_PRIKEY_PKCS8_UNENCRYPT;
   uint32_t dkFormat = CRYPT_ALGO_MLDSA_PRIV_FORMAT_PRIV_ONLY;
   int32_t ret = CRYPT_EAL_PkeyCtrl(pkey, CRYPT_CTRL_SET_MLDSA_PRVKEY_FORMAT, &dkFormat, sizeof(dkFormat));
   // path参数指明编码密钥的文件路径
   ret = CRYPT_EAL_EncodeFileKey(pkey, NULL, format, type, "mldsa65_priv_only.pem");
   ```

   ML-DSA密钥对编码前可以完成私钥编码表示形式的设置，编码格式需与密钥格式保持一致。具体通过调用`CRYPT_EAL_PkeyCtrl`接口并传入`CRYPT_CTRL_SET_MLDSA_PRVKEY_FORMAT`命令以及设置的编码格式枚举。编码格式由`CRYPT_ALGO_MLDSA_PRIV_KEY_FORMAT_TYPE`枚举指定，共支持以下三种形式：

   + **CRYPT_ALGO_MLDSA_PRIV_FORMAT_BOTH**：包括生成私钥所需种子和展开后私钥数据的完整格式
   + **CRYPT_ALGO_MLDSA_PRIV_FORMAT_PRIV_ONLY**：仅包含展开后私钥数据的格式
   + **CRYPT_ALGO_MLDSA_PRIV_FORMAT_SEED_ONLY**：仅包含生成私钥所需种子的格式

   **密钥解码**

   ```c
   // 1. 从缓冲区解码密钥对
   int32_t ret = CRYPT_EAL_PkeyCtx *prvKey = NULL;
   // 设置format和type，keyBuf为存储密钥对编码的缓冲区
   ret = CRYPT_EAL_DecodeBuffKey(BSL_FORMAT_PEM, CRYPT_PRIKEY_PKCS8_UNENCRYPT, &keyBuf, NULL, 0, &prvKey);
   
   // 2. 从文件解码密钥对
   int32_t ret = CRYPT_EAL_PkeyCtx *prvKey = NULL;
   ret = CRYPT_EAL_DecodeBuffKey(BSL_FORMAT_PEM, CRYPT_PRIKEY_PKCS8_UNENCRYPT, "mldsa65_priv_only.pem", NULL, 0, &prvKey);
   ```

### 证书设置

1. **证书创建和解析**

   ```c
   // 创建一个新证书

   if (cert == NULL) {
       printf("创建空证书对象失败\n");

   }
   
   // 从文件中读取证书
   HITLS_X509_Cert *cert = NULL;
   HITLS_X509_CertParseFile(BSL_FORMAT_PEM, "mldsa_end.crt", &cert);
   if (cert == NULL) {
       printf("创建空证书对象失败\n");
       return HITLS_X509_ERR_CERT_EXIST;
   }
   ```

   

2. **证书设置**

   ```c
   // 设置证书相关信息并配置公钥
   ret = HITLS_X509_CertCtrl(cert, HITLS_X509_SET_VERSION, &version, sizeof(version));
   ret = HITLS_X509_CertCtrl(cert, HITLS_X509_SET_SERIALNUM, serialNum, sizeof(serialNum));
   ···
   ret = HITLS_X509_CertCtrl(cert, HITLS_X509_SET_PUBKEY, pubkey, 0);
   
   // 可按需求定义KeyUsage 扩展
   HITLS_X509_ExtKeyUsage ku;
   ku.critical = true;
   ku.keyUsage = HITLS_X509_EXT_KU_KEY_CERT_SIGN;
   ret = HITLS_X509_CertCtrl(cert, HITLS_X509_EXT_SET_KUSAGE, &ku, sizeof(HITLS_X509_ExtKeyUsage));
   ```

   如果后量子证书的公钥算法是ML-DSA算法或SLH-DSA算法，需要对其密钥用途进行限制。KeyUsage扩展需至少包含以下其中一个用途：

   + `HITLS_X509_EXT_KU_DIGITAL_SIGN`（数字签名）
   + `HITLS_X509_EXT_KU_NON_REPUDIATION`（不可否认签名）
   + `HITLS_X509_EXT_KU_KEY_CERT_SIGN`（签发证书）
   + `HITLS_X509_EXT_KU_CRL_SIGN`（签发CRL）

   并且禁止以下所有用途：

   + `HITLS_X509_EXT_KU_KEY_ENCIPHERMENT`（密钥加密）
   + `HITLS_X509_EXT_KU_DATA_ENCIPHERMENT`（数据加密）
   + `HITLS_X509_EXT_KU_KEY_AGREEMENT`（密钥协商）
   + `HITLS_X509_EXT_KU_ENCIPHER_ONLY`（仅加密）
   + `HITLS_X509_EXT_KU_DECIPHER_ONLY`（仅解密）

3. **证书签名与验证**

   ```c
   // 签发ML-DSA证书
   ret = HITLS_X509_CertSign(CRYPT_MD_SHA256, signKey, &algParam, cert);
   
   ret = HITLS_X509_CertGenFile(BSL_FORMAT_PEM, cert, "mldsa.pem");
   
   // 1. 使用公钥进行验证
   ret = HITLS_X509_CertVerifyByPubKey(cert, pubkey);
   
   // 2. 构建证书链进行验证
   // 定义存储上下文和证书链指针
   HITLS_X509_StoreCtx *store = HITLS_X509_StoreCtxNew();
   HITLS_X509_List *chain = NULL;
   
   // 加入中间CA证书到上下文
   HITLS_X509_Cert *inter = NULL;
   ret = HITLS_X509_CertParseFile(BSL_FORMAT_UNKNOWN, "mldsa_inter.pem", &inter);
   ret = HITLS_X509_StoreCtxCtrl(store, HITLS_X509_STORECTX_DEEP_COPY_SET_CA, inter, sizeof(HITLS_X509_Cert));
   if (ret != HITLS_PKI_SUCCESS) {
       return ret;
   }
   // 解析终端证书
   HITLS_X509_Cert *entity = NULL;
   ret = HITLS_X509_CertParseFile(BSL_FORMAT_PEM, "mldsa_entity.pem", &entity);
   
   // 构建证书链
   ret = HITLS_X509_CertChainBuild(store, false, entity, &chain);
   
   // 将根证书加入上下文
   HITLS_X509_Cert *root = NULL;
   ret = HITLS_X509_CertParseFile(BSL_FORMAT_PEM, "mldsa_root.pem", &root);
   
   ret = HITLS_X509_StoreCtxCtrl(store, HITLS_X509_STORECTX_DEEP_COPY_SET_CA, root, sizeof(HITLS_X509_Cert));
   
   // 重新构建证书链
   ret = HITLS_X509_CertChainBuild(store, false, entity, &chain);
   
   // 验证证书链
   ret = HITLS_X509_CertVerify(store, chain);
   ```

4. **CMS签名数据验证**

   ```c

   HITLS_X509_Cert *caCert = NULL;
   HITLS_X509_List *caCertList = NULL;
   // 要读取的文件路径，分别存放CMS文件、消息数据、CA证书
   char *path = "mldsa65_sha384_attached.cms";
   char *msg = "msg.txt";
   char *ca_cert = "ca_cert.pem";

   ret = HITLS_X509_CertParseFile(BSL_FORMAT_PEM, caPath, &caCert);

   ret = BSL_LIST_AddElement(caCertList, caCert, BSL_LIST_POS_END);
   // 将CA证书列表封装成统一的参数结构
   BSL_Param params[2] = {
       {HITLS_CMS_PARAM_CA_CERT_LISTS, BSL_PARAM_TYPE_CTX_PTR, caCertList, 0, 0},
       BSL_PARAM_END
   };
   // 调用CMS签名数据验证接口
   ret = HITLS_CMS_DataVerify(cms, &msgBuff, params, NULL);
   ```

   ML-DSA算法在CMS验证流程中需要与哈希算法适配，当CMS签名数据使用了不符合ML-DSA参数安全要求的哈希算法时，验证接口必须拒绝验证。具体参照[HITLS_CMS_DataSign](#hitls_cms_datasign)一节中的对应关系。

   | `HITLS_CMS_ERR_MLDSA_INVALID_DIGEST`   | MLDSA算法与摘要算法对应关系不符合RFC9882要求 | `HITLS_CMS_ProviderParseFile`解析出的CMS错误或创建CMS调用`HITLS_CMS_DataSign`函数中`pkey`和`params`中的算法不匹配 |
   | `HITLS_CMS_ERR_MLDSA_ERROR_DIGEST`     | ML-DSA算法摘要算法不符合要求                 | CMS结构在不包含signedAttrs时摘要算法不为SHA512在`HITLS_CMS_DataVerify`调用过程中报错 |
   | `HITLS_CMS_ERR_PQC_PARAMS_NOT_OMITTED` | PQC算法参数在算法标识中未被省略              | 调用`HITLS_CMS_ProviderParseBuff/File`函数时解析CMS结构出现了不应出现的显式参数 |
   | `HITLS_CMS_ERR_NOT_SUPPORT_STREAM_PQC` | PQC算法不支持流式签名/验签                   | 使用PQC算法调用`HITLS_CMS_DataInit`和`HITLS_CMS_DataUpdate`流式签名最终在调用`HITLS_CMS_DataFinal`时报错 |
   
   

## SLH-DSA使用指南

### 初始设置
与[MLKEM](#初始设置)中的设置保持一致。

### 密钥设置

1. **创建SLH-DSA密钥对**

   ```c
   // 根据SLH-DSA对应的算法ID创建非对称密钥上下文
   CRYPT_EAL_PkeyCtx *ctx = CRYPT_EAL_PkeyNewCtx(CRYPT_PKEY_SLH_DSA);
   // 根据参数id设置参数，参数id可选
   uint32_t id = CRYPT_SLH_DSA_SHA2_128S;
   int32_t ret = CRYPT_EAL_PkeySetParaById(ctx, id);
   // 初始化随机数
   ret = CRYPT_EAL_ProviderRandInitCtx(NULL, CRYPT_RAND_SHA256, "provider=default", NULL, 0, NULL);
   // 生成密钥对保存在上下文中
   ret = CRYPT_EAL_PkeyGen(ctx);
   ```

   SLH-DSA算法支持的参数id包括SHA2/SHAKE族在128/192/256安全等级下的Small与Fast两类参数集，具体包括如下表所示。

   | 算法标识                   | 哈希类型 | 安全级别 | 参数集 |
   | -------------------------- | -------- | -------- | ------ |
   | `CRYPT_SLH_DSA_SHA2_128S`  | SHA-2    | 128 bit  | Small  |
   | `CRYPT_SLH_DSA_SHAKE_128S` | SHAKE    | 128 bit  | Small  |
   | `CRYPT_SLH_DSA_SHA2_128F`  | SHA-2    | 128 bit  | Fast   |
   | `CRYPT_SLH_DSA_SHAKE_128F` | SHAKE    | 128 bit  | Fast   |
   | `CRYPT_SLH_DSA_SHA2_192S`  | SHA-2    | 192 bit  | Small  |
   | `CRYPT_SLH_DSA_SHAKE_192S` | SHAKE    | 192 bit  | Small  |
   | `CRYPT_SLH_DSA_SHA2_192F`  | SHA-2    | 192 bit  | Fast   |
   | `CRYPT_SLH_DSA_SHAKE_192F` | SHAKE    | 192 bit  | Fast   |
   | `CRYPT_SLH_DSA_SHA2_256S`  | SHA-2    | 256 bit  | Small  |
   | `CRYPT_SLH_DSA_SHAKE_256S` | SHAKE    | 256 bit  | Small  |
   | `CRYPT_SLH_DSA_SHA2_256F`  | SHA-2    | 256 bit  | Fast   |
   | `CRYPT_SLH_DSA_SHAKE_256F` | SHAKE    | 256 bit  | Fast   |

   **典型错误枚举**：

   | `CRYPT_SLHDSA_ERR_INVALID_ALGID`        | 未设置参数集或参数集超出支持范围    | 未调用`CRYPT_EAL_PkeySetParaById`函数或是传入了无效的参数    |

   ```c
   HITLS_X509_StoreCtx *store = HITLS_X509_StoreCtxNew();
   HITLS_X509_List *chain = NULL;
   
   // 加入中间CA证书到上下文
   HITLS_X509_Cert *inter = NULL;

   HITLS_X509_Cert *entity = NULL;
   ret = HITLS_X509_CertParseFile(BSL_FORMAT_PEM, "slhdsa_entity.pem", &entity);
   
   // 构建证书链
   ret = HITLS_X509_CertChainBuild(store, false, entity, &chain);

   // 重新构建证书链
   ret = HITLS_X509_CertChainBuild(store, false, entity, &chain);
   ```

## XMSS使用指南

### 初始设置
与[MLKEM](#初始设置)中的设置保持一致。

### 密钥设置

1. **创建XMSS密钥对**

   ```c
   // 根据XMSS对应的算法ID创建非对称密钥上下文
   CRYPT_EAL_PkeyCtx *ctx = CRYPT_EAL_PkeyNewCtx(CRYPT_PKEY_XMSS);
   // 根据参数id设置参数
   uint32_t id = CRYPT_XMSS_SHA2_10_256;
   int32_t ret = CRYPT_EAL_PkeySetParaById(ctx, id);
   // 初始化随机数
   ret = CRYPT_EAL_ProviderRandInitCtx(NULL, CRYPT_RAND_SHA256, "provider=default", NULL, 0, NULL);
   // 生成密钥对保存在上下文中
   ret = CRYPT_EAL_PkeyGen(ctx);
   ```

   XMSS算法支持的参数id包括XMSS与XMSSMT的SHA2、SHAKE、SHAKE256变种，在不同树高度、层数及安全参数下的全部标准化参数组合，其中除了哈希类型和安全级别以外，XMSS参数指定了Merkle树的高度，XMSSMT参数指定了层数和总的树高度，具体取值参照`CRYPT_PKEY_ParaId`中包含`XMSS`的枚举。

   **典型错误枚举**：

   | 枚举值                         | 错误原因                       | 触发场景                                                  |
   | ------------------------------ | ------------------------------ | --------------------------------------------------------- |
   | `CRYPT_XMSS_ERR_INVALID_ALGID` | XMSS参数集非法或不在支持范围内 | 未调用`CRYPT_EAL_PkeySetParaById`函数或是传入了无效的参数 |

2. **密钥控制**

   ```c
   // 获取XMSS算法XDR算法类型
   uint8_t xdr[4] = {0};
   int32_t ret = CRYPT_EAL_PkeyCtrl(pctx, CRYPT_CTRL_GET_XMSS_XDR_ALG_TYPE, xdr, sizeof(xdr));
   
   // 设置XMSS算法XDR算法类型，RFC9802定义XDR算法表示长度为4字节
   int32_t ret = CRYPT_EAL_PkeyCtrl(pctx, CRYPT_CTRL_SET_XMSS_XDR_ALG_TYPE, xdr, 4);
   ```

3. **密钥对的编解码**

   **密钥编码**

   ```c
   // 按照创建密钥对的方式得到非对称密钥上下文pkey后用来编码
   
   // 1. 编码到缓冲区
   // 初始化缓冲区
   BSL_Buffer out = {0};
   // 调用编码接口，设置format和type参数，得到存放编码的缓冲区out
   ret = CRYPT_EAL_EncodeBuffKey(pkey, NULL, BSL_FORMAT_PEM, CRYPT_PRIKEY_PKCS8_UNENCRYPT, &out);
   
   // 2. 编码到文件
   int32_t format = BSL_FORMAT_PEM;
   int32_t type = CRYPT_PRIKEY_PKCS8_UNENCRYPT;
   // path参数指明编码密钥的文件路径
   ret = CRYPT_EAL_EncodeFileKey(pkey, NULL, format, type, "xmss_sha2_10_256.pem");
   ```

   **密钥解码**

   ```c
   // 1. 从缓冲区解码密钥对
   int32_t ret = CRYPT_EAL_PkeyCtx *prvKey = NULL;
   // 设置format和type，keyBuf为存储密钥对编码的缓冲区
   ret = CRYPT_EAL_DecodeBuffKey(BSL_FORMAT_PEM, CRYPT_PRIKEY_PKCS8_UNENCRYPT, &keyBuf, NULL, 0, &prvKey);
   
   // 2. 从文件解码密钥对
   int32_t ret = CRYPT_EAL_PkeyCtx *prvKey = NULL;
   ret = CRYPT_EAL_DecodeBuffKey(BSL_FORMAT_PEM, CRYPT_PRIKEY_PKCS8_UNENCRYPT, "xmss_sha2_10_256.pem", NULL, 0, &prvKey);
   ```

   **设置公私钥**

   ```c
   // 设置公钥
   CRYPT_EAL_PkeyPub pub = { 0 };
   // 设置算法类型
   pub.id = CRYPT_PKEY_XMSS;
   // 设置公钥参数
   pub.key.xmssPub.seed = key->x;
   pub.key.xmssPub.root = key->x + keyLen;
   pub.key.xmssPub.len = keyLen;
   // 调用函数将公钥结构存放如入上下文
   ret = CRYPT_EAL_PkeySetPub(pkey, &pub);
   
   // 设置私钥
   CRYPT_EAL_PkeyPrv prv = { 0 };
   // 设置算法类型
   prv.id = CRYPT_PKEY_XMSS;
   // 设置seed等私钥参数，所有参数存储在十六进制串key中
   prv.key.xmssPrv.index = index;
   prv.key.xmssPrv.seed = prvSeed;
   prv.key.xmssPrv.prf = prvPrf;
   prv.key.xmssPrv.pub.seed = pubSeed;
   prv.key.xmssPrv.pub.root = pubRoot;
   prv.key.xmssPrv.pub.len = keyLen;
   // 调用函数将私钥结构存放如入上下文
   ret = CRYPT_EAL_PkeySetPrv(pkey, &prv);
   ```
